Assessing the Impact of Code Samples Evolution on Developers' Doubts

Abstract. To accelerate the learning process of technical resources by developers, code samples have been provided and maintained by organizations, such as Amazon, Google, and Microsoft. Code samples are special software projects made available in public repositories with the main goal of teaching developers how to use features of frameworks, libraries, or other technical resources. Since code samples are software projects themselves, they evolve over time. However, if changes do not preserve essential characteristics such as size and simplicity throughout evolution, code samples may lose their usefulness. In this work, we investigate how code samples evolve and how evolution may impact on developers questions. We conduct an exploratory study on the evolution of 159 code samples provided by Android, AWS, Azure, and Spring Boot, assessing metrics related to size and complexity as well as developers questions about the analyzed code samples in Stack Overflow. We observed that organizations may not be paying due attention to the key characteristics of code samples during evolution. Our analysis also indicates that changes in some of these characteristics are related to an increase in questions about code samples.
